Тестовое задание, интеграция в amoCRM.

1) Реализовать эндпоинт который будет обращаться в amoCRM к сделкам на этапе "Заявка" в воронке "Воронка", перебирать все найденные сделки и проверять, если у сделки бюджет > 5000, то он перетягивает ее на этап "Ожидание клиента"

2) Реализовать эндпоинт, который будет копировать сделки на этапе “Клиент подтвердил” при условии, что бюджет сделки = равен 4999 

Необходимые действия для реализации:
a) Обратиться к сделкам на этапе “Клиент подтвердил” и выбрать только те, у которых бюджет равен 4999.
b) Обратиться к примечаниям сделок, которые были найдены после перебора.
c) Обратиться к задачам сделок, которые были найдены после перебора.
d) Создать копию сделки с переносом всех значений в полях. Новую сделку нужно создавать на этапе "Ожидание клиента".
e) Создать копии примечаний из сделки-донора и прикрепить новые примечания к новой сделке.
f) Создать копии задач из сделки-донора с сохранением всех данных и прикрепить новые задачи к сделке.

Параметры для аутентификации в амо, а также ID воронок и этапов нужно указывать в config/amocrm_config.php.

Реализована автозагрузка классов в стандарте PSR-4

Доступные эндпоинты:

1. /index.php?action=move - Переместить сделки с бюджетом > 5000

2. /index.php?action=copy - Скопировать сделки с бюджетом = 4999(также копируются задачи и примечания из сделок)
